# Project Log Skill — 프로젝트 진행 기록 비서

## 기록 원칙
- 기록 파일은 **각 프로젝트 안에** 둔다: `PROJECT_LOG.md`
- 본부(jackson-workspace)에 통합하지 않는다 — 그 방 AI가 바로 확인하고 이어갈 수 있게.

---

## 작업 시작 시
`PROJECT_LOG.md`가 있으면 읽어서 이전 기록을 확인하고 이어간다.
없으면 새로 만든다.

---

## 작업 마무리 시 — 항상 실행

### 1) PROJECT_LOG.md 업데이트

아래 형식으로 기록:

```markdown
## [날짜] [성공/실패]
### 작업 내용
- 무엇을 했는가 한 줄

### 결과
- 성공: 무엇이 됐는가
- 실패: 무엇이 안 됐는가, 원인 추정

### 배운 것 / 반복하면 안 되는 실수
- (있으면 기록)

### 성공 루틴 (재사용 가능한 방법)
- (있으면 기록)

### 배포 이력
- 버전: vX.X.X → vX.X.X
- 배포 일시: [날짜 시간]
```

### 2) git 자동 처리 (완전 자동, 확인 없이)

```bash
# 버전업 (package.json 있으면 patch 자동 bump, 없으면 git tag로 관리)
# 커밋
git add .
git commit -m "[날짜] [작업 한 줄 요약]"
# 푸시
git push
# 배포 (프로젝트별 배포 명령 있으면 실행)
```

완료 후 사용자에게 한 줄 보고:
> `배포완료!! v[버전] | [커밋 메시지 한 줄]`

---

## 반복 실수 방지
- 작업 시작 전 `PROJECT_LOG.md`의 "반복하면 안 되는 실수" 항목을 읽는다.
- 같은 실수가 기록에 있으면 시작 전에 사용자에게 알린다.

---

## 새 프로젝트 시작 시
`PROJECT_LOG.md`를 프로젝트 루트에 생성:
```markdown
# [프로젝트명] 진행 기록
생성일: [날짜]
---
```
